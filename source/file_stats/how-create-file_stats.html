模块概览

file_stats 在本地文件系统上提供：元数据读取、MD5 / SHA-1 / SHA-256 哈希、双文件对比、checksum 校验、目录统计（按扩展名 breakdown）、重复文件（先按 size 分桶再 SHA-256）、大文件扫描、按文件名关键字搜索、UTF-8 文本统计、批量路径信息。

manifest：capabilities: ["file_read"]；需 MANAGE_EXTERNAL_STORAGE。pluginClass：ai.rorsch.moduleplugins.file_stats.FileStatsPlugin。

架构

  • 入口：invoke → JSONObject params → 各私有方法返回业务 JSON 字符串 → ok(output, _displayText) 包装。
  • 哈希：computeHash(File, algorithm) 使用 MessageDigest，8KB 缓冲流式读取；normalizeAlgo 将用户输入规范为 MD5 / SHA-1 / SHA-256（默认）。
  • 目录遍历：collectDirStats、collectFilesBySize、collectLargeFiles、searchByNameRecursive 等 DFS。
  • 展示：多种 formatXxxDisplay 静态方法生成 Markdown 表格风格 _displayText，列表最多 DISPLAY_LIST_MAX = 20 条。

按功能分组的方法

getFileInfo / buildFileInfo

buildFileInfo(File) 输出 path、name、isDirectory、size、lastModified、权限、hidden、extension、mimeType（MimeTypeMap）、目录时 childCount。

getFileHash

目录不可哈希；返回 path、algorithm、hash、size。

compareFiles

先比 length；相同则两文件各算 SHA-256，填 identical、reason。

verifyChecksum

actual 与 expectedHash 忽略大小写比较 → verified。

getDirStats / collectDirStats

counts[0,1,2] = totalSize、fileCount、directoryCount；extensionBreakdown 按总 size 降序。

findDuplicates / collectFilesBySize

同 size 且 ≥2 才读文件算 SHA-256；wastedSpace = 每组 (count-1)×fileSize 之和。

findLargeFiles / collectLargeFiles

默认 minSizeMB=100，limit=50，结果按 size 降序截断。

searchByName

keywords 逗号分隔，转小写；文件名 contains 任一 keyword；limit 默认 100。

getTextStats

UTF-8 逐行；characters 为每行 length+1 近似换行。

batchFileInfo

paths 逗号分隔；存在则元数据，文件则带 hash。
