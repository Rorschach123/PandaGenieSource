上层 Agent / Task 通过 action 字符串与 JSON params 调用插件；插件将结果序列化为 JSON，其中成功时常包含人类可读的 _displayText（含 emoji）以便 UI 展示。

架构：四层 Architecture（4-Layer）

自顶向下：UI / 自动化编排（可为 HTML 或 App 内 WebView）→ Java ModulePlugin → JNI → C++ calc::Calculator。

Layer 1 — Presentation / Orchestration调用方构造 action + paramsJson，解析返回 JSON（success、output、可选 _displayText）。
Layer 2 — Java PluginCalculatorPlugin：校验 action，从 JSONObject 读取参数，调用 CalculatorLib 的 native 方法。
Layer 3 — JNI Bridgecalculator_jni.cpp：C 链接名 Java_ai_rorsch_pandagenie_nativelib_CalculatorLib_*，将 Java 类型映射到 calc::Calculator；错误路径映射为 ArithmeticException 或 RuntimeException。
Layer 4 — Native Corecalculator.cpp：基于 <cmath> 的数学实现；ExpressionParser 递归下降解析表达式字符串。

辅助方法 Helpers

  • emptyJson(String value)：null 或空白 → "{}"，避免 JSONObject 构造失败。
  • formatParam(double value) / formatNumber(double value)：若值为整型外观（value == (long)value），输出无小数；否则 String.valueOf(double)。
  • ok(String output)：success=true，output。
  • ok(String output, String displayText)：额外写入非空 _displayText。
  • error(String message)：success=false，error 字段。

C++ Native Layer

calculator.h（当前约 93 行）

  • Namespace：calc。
  • Class Calculator：静态方法涵盖基础运算、power/sqrt/cbrt/nthRoot、三角（弧度）、atan2、双曲、对数（含 logBase）、abs/ceil/floor/round、阶乘与排列组合、角度换算、常量 pi()/e()、evaluate(const std::string&)、listFunctions()。
  • Class ExpressionParser（recursive-descent parser）：
    • 字段：expr_（表达式字符串）、pos_（当前读取下标）。
    • 公开：explicit ExpressionParser(const std::string& expr)、double parse()。
    • 私有：parseExpression、parseTerm、parseUnary、parsePower、parseAtom、parseFunction、parseNumber、peek、get、skipWhitespace、match。

calculator.cpp（当前约 308 行）

  • 数学实现统一使用 <cmath>。
  • Error handling（抛 std::runtime_error）：
    • divide / modulo：除数为零。
    • sqrt：x < 0。
    • nthRoot：n==0；偶次根号下为负。
    • asin / acos：x 不在 [-1,1]。
    • ln / log10 / log2：x <= 0；logBase 额外校验底数。
    • factorial：n < 0；n > 170 视为 overflow。
    • permutation / combination：n,r 非法（负或 r > n）。
  • ExpressionParser 递归下降：
    • 结合性与优先级（从低到高）：parseExpression（+ -）→ parseTerm（* / %，操作数为 parseUnary）→ parseUnary（一元 +/-）→ parsePower（^，右侧为 parseUnary）→ parseAtom（数字、括号、常量 pi/e、函数调用）。
    • 函数名大小写不敏感（标识符读入后 tolower）。
    • 支持的函数（节选与插件文档对应）：sin、cos、tan、asin、acos、atan、atan2、sinh、cosh、tanh、sqrt、cbrt、abs、ceil、floor、round、ln、log（实现为 log10）、log10、log2、pow/power（二参）、mod、max、min、deg/rad（角度换算）、fact、perm、comb。
    • 常量：pi、e 映射 M_PI、M_E。
    • 解析结束：若仍有未消费字符则 Unexpected character。

calculator_jni.cpp（当前约 97 行）

#define JNI_METHOD(ret, name) \
    extern "C" JNIEXPORT ret JNICALL \
    Java_ai_rorsch_pandagenie_nativelib_CalculatorLib_##name
  • 每个 JNI 函数体直接调用 calc::Calculator::...；nSqrt → Calculator::sqrt。
  • try-catch + ThrowNew ArithmeticException：divide、modulo、nSqrt、asin、acos、ln、log10、log2、factorial、permutation、combination（与源码一致）。
  • evaluate：GetStringUTFChars → std::string → Calculator::evaluate；异常映射为 java/lang/RuntimeException（与 ArithmeticException 区分）。
  • listFunctions：Calculator::listFunctions() → NewObjectArray + NewStringUTF 填充 String[]。

表达式解析器文法（BNF 风格）

下列描述与 calculator.cpp 中 ExpressionParser 实现一致；WS 表示空白，在 token 间任意出现。

expression   ::= WS term WS ( ('+' | '-') WS term WS )*
term         ::= WS unary WS ( ('*' | '/' | '%') WS unary WS )*
unary        ::= WS ('+' | '-')? WS power
power        ::= WS atom WS ( '^' WS unary )?

atom         ::= number
              | '(' WS expression WS ')'
              | identifier
identifier   ::= name WS ( '(' WS arglist? WS ')' )?
              | 'pi' | 'e'          // 无括号常量（在实现中与函数名统一为小写匹配）

arglist      ::= expression ( ',' WS expression )*

number       ::= digit+ ( '.' digit* )? | '.' digit+

name         ::= [A-Za-z_][A-Za-z0-9_]*   // 解析时转为小写

// 内置函数名（name 部分，小写）包括但不限于：
// sin cos tan asin acos atan atan2 sinh cosh tanh
// sqrt cbrt abs ceil floor round
// ln log log10 log2
// pow power mod max min deg rad fact perm comb

运算符优先级（从低到高）：加减 → 乘除取模 → 一元正负 → 幂 ^（右操作数为 unary）→ 原子（数字/括号/常量/函数）。
