模块概述

经典 Tic-Tac-Toe（井字棋）：玩家执 X 且先手，程序执 O。棋盘为 3×3，内部用整数矩阵：EMPTY=0，PLAYER_X=1，PLAYER_O=2。placeMark 落子后立即调用启发式 aiMove()，再检测胜负或平局。

胜负判定：winner 内部值 0=none，1=玩家胜，2=AI 胜，3=平局(draw)；buildState() 映射为字符串 "none"|"player"|"ai"|"draw"。

棋盘编码与 AI 策略（minimax-style 描述）

源码注释为「Try to win / Block / Center / Corner / Any」，属于浅层必胜/必堵搜索而非完整博弈树：

  1. findBestMove(player)：遍历每个空格，临时放置该 player，若 checkWin(player) 为真则该步为「制胜步」，恢复棋盘后返回坐标。
  2. AI 先手顺序：若存在 O 的制胜步则下；否则若存在 X 的制胜步则堵；否则若中心空则占中心；否则在四个角中随机选一空角；否则任选一空格。

checkWin 检查三行、三列与两条对角线。

API Actions — 输入 / 输出

startGame

params：无

output：初始空棋盘状态。

placeMark

params：row，col（0–2）

已被占用或越界返回 error；游戏已结束则返回当前状态（success）不修改棋盘。

getState · restart

getState 需游戏已开始。restart 保证 random 非空后 initGame()。

数据流

startGame → initGame
placeMark:
  board[r][c]=X → checkWinner
       → if !gameOver: aiMove(O) → checkWinner
       → ok(buildState, formatDisplay)

胜负: checkWin 行/列/对角 → winner 1 或 2
平局: 棋盘满且无人胜 → winner=3 (draw)
