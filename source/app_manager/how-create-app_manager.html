模块概述

本模块为 已安装应用管理（app_manager），基于 Android PackageManager 与 Intent 系统，提供列举应用、启动应用、查询详情、发起卸载请求、打开系统「应用信息」设置页等能力。适用于 Agent / 任务在已授权环境下调用。

权限： android.permission.REQUEST_DELETE_PACKAGES（用于请求卸载确认对话框）。Capability： app_list。

插件类： ai.rorsch.moduleplugins.app_manager.AppManagerPlugin，实现接口 ModulePlugin，由宿主 ModuleRuntime 反射加载并调用 invoke(Context, action, paramsJson)。

架构与数据流

模块 H5 页面（index.html）通过 JavaScript 调用宿主注入的 NativeBridge，将 action 与 JSON 参数传入原生侧；原生 ModuleRuntime 将请求路由至 AppManagerPlugin，执行 PackageManager 查询或 startActivity，返回统一 JSON 包装。

用户 / AI Task → WebView 加载 index.html → JS: NativeBridge.invoke(action, JSON.stringify(params)) → 原生 ModulePlugin.invoke(context, action, paramsJson) → AppManagerPlugin: PackageManager / Intent / Settings ← JSON: { success, output, _displayText? } 或 { success:false, error }
H5 与 Plugin 对应关系（节选）
  • listApps：列表页加载、含「显示系统应用」复选框 → includeSystem
  • getAppInfo：点击条目打开详情 Modal
  • openApp / openAppSettings / uninstallApp：按钮触发

NativeBridge 调用约定

前端典型调用：NativeBridge.invoke(action, JSON.stringify(params))，解析返回对象 res：res.success === false 时读取 res.error；成功时业务数据在 res.output（字符串，多为 JSON 字符串），可选 res._displayText 供聊天 UI 展示摘要。

// 示例（与 index.html 一致）
var raw = NativeBridge.invoke("openApp", JSON.stringify({ nameOrPackage: "微信" }));
var res = JSON.parse(raw);

API Actions — 输入 / 输出

listApps
输入参数类型说明
includeSystemboolean是否包含系统应用，默认 false

output（JSON 字符串内嵌于 output）：count（int），apps（数组，元素含 packageName, appName, versionName），按 appName 忽略大小写排序。

实现要点： pm.getInstalledPackages(0)，默认跳过 isSystemApp 为 true 的包。

openApp
参数说明
nameOrPackage必填，应用显示名或完整包名

成功 output： launched, packageName, appName。失败： 未找到包、无 LAUNCH Activity 等。

使用 getLaunchIntentForPackage + FLAG_ACTIVITY_NEW_TASK。

getAppInfo
参数说明
nameOrPackage必填

output 字段： packageName, appName, versionName, versionCode, firstInstallTime, lastUpdateTime（格式化字符串）, targetSdkVersion, minSdkVersion, isSystemApp, enabled, dataDir, sourceDir, installerPackage, installerLabel。

uninstallApp
参数说明
packageName优先；若空则尝试 nameOrPackage 经 resolvePackageName 解析

output： requestSent, packageName, message。启动 Intent.ACTION_DELETE，package: URI。

openAppSettings
参数说明
nameOrPackage必填

使用 Settings.ACTION_APPLICATION_DETAILS_SETTINGS + package: URI。

output： opened, packageName。

Java 类与方法详解

类： public class AppManagerPlugin implements ModulePlugin

invoke(Context context, String action, String paramsJson)

入口：将 paramsJson 经 emptyJson 解析为 JSONObject，按 action 分发。listApps / getAppInfo 在成功后附加 _displayText（Markdown 风格表格字符串）。未知 action 返回 error("Unsupported action: ...")。

私有业务方法
  • listApps(Context, JSONObject) — 构建应用列表 JSON。
  • openApp(Context, JSONObject) — 解析包名并启动。
  • getAppInfo(Context, JSONObject) — getPackageInfo(packageName, 0) 填充详情。
  • uninstallApp(Context, JSONObject) — 卸载意图。
  • openAppSettings(Context, JSONObject) — 应用详情设置页。
关键算法：resolvePackageName / isSystemApp / resolveInstallerLabel

resolvePackageName(context, nameOrPkg)： 先 getPackageInfo(nameOrPkg, 0) 精确匹配包名；失败则遍历已安装包：优先显示名 equalsIgnoreCase；否则取第一个显示名（小写）包含输入小写子串的匹配（首个命中）。无则 null。

isSystemApp(PackageInfo pi)： 检查 ApplicationInfo.FLAG_SYSTEM。

resolveInstallerLabel(String installerPkg)： 将 getInstallerPackageName 结果映射为可读标签（Google Play、华为、小米、OPPO、vivo、应用宝、三星等），未知则返回包名或 "Unknown"。

展示格式化与响应包装
  • emptyJson(String v) — 空或空白 → "{}"。
  • mdCell(String s) — 表格单元转义（换行、管道符）。
  • formatListAppsDisplay(JSONObject) / formatGetAppInfoDisplay(JSONObject) — 生成 _displayText。
  • appendRow(StringBuilder, item, value) — 辅助表格行。
  • ok(String output) / ok(String output, String displayText) — success: true。
  • error(String message) — success: false, error 字段。
