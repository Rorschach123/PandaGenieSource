模块概述

在设备端离线完成常见 text processing：wordCount（含 CJK 分词规则）、Base64 / URL 编解码、Java Pattern 正则匹配与替换、大小写与反转等变换、UUID 随机生成、MD5/SHA-1/SHA-256 摘要。依赖 android.util.Base64 与 java.security.MessageDigest。

架构与算法要点

  • 字数统计：characters 使用 String.codePointCount（Unicode 码点）。countWords：Character.isIdeographic 的码点各计 1 词；否则以连续非空白且非表意片段为英文「词」。
  • 行数：空字符串视为 1 行；统计 \n、\u2028、\u2029 及 \r\n。
  • 句数：正则 (?<=[\.\!\?。！？])(?=\s|$) 切分后统计非空片段，至少为 1（非空 trim 文本）。
  • 段数：双换行 \R\s*\R 分段，空文本 0 段，否则至少 1。
  • 正则：pattern 为空抛 IllegalArgumentException；语法错误包装为 invalid regex。
  • 哈希：算法名大小写不敏感；SHA1/SHA256 映射到 SHA-1/SHA-256；输出小写十六进制。

API Actions — 输入 / 输出

wordCount

params：text

output：characters，words，lines，sentences，paragraphs

base64Encode / base64Decode

params：text

output：encoded 或 decoded

urlEncode / urlDecode

说明：application/x-www-form-urlencoded，空格编码为 +。

output：encoded / decoded

regexMatch

params：text，pattern（必填）

output：matches（JSON 数组，每项为整段匹配字符串），count

regexReplace

params：text，pattern，replacement

output：result（全文替换后）

textTransform

params：text，transform ∈ {upper,lower,capitalize,reverse,trim}（大小写不敏感）

output：result

generateUUID

params：无

output：uuid（RFC 4122 随机 UUID 字符串）

hashText

params：text，algorithm（MD5 | SHA-1 | SHA-256）

output：algorithm（规范名），hex

数据流

paramsJson → JSONObject
    → switch(action)
        → 业务方法返回 JSONObject 或 String
        → ok(outputString, _displayText)  // 多数 action 的 output 为紧凑 JSON 字符串

异常路径: catch → error(exception.message)

展示: _displayText 为 Markdown 表格字符串，供 Chat UI 渲染
