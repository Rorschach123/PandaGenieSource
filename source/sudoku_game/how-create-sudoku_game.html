模块概述

本模块实现经典 Sudoku（数独）：在 9×9 网格中填入数字 1–9，使每行、每列及每个 3×3 子宫（box） 内数字不重复。题目由先完整生成合法终盘（solution），再按难度随机挖空若干格得到 puzzle；玩家盘面为 playerBoard，预填格由 fixed[][] 标记且不可修改或清除。

难度（difficulty）：取值 1–3，对应挖空数量：35（简单）、45（普通，默认）、55（困难）格。

宿主集成：实现 ModulePlugin，入口 invoke(Context, action, paramsJson)；成功返回 success=true，业务 JSON 置于字符串字段 output（二次序列化），可选 _displayText 供 UI 展示。

架构与数据模型

状态矩阵与常量
  • SIZE = 9、BOX = 3
  • int[][] solution：完整终盘答案。
  • int[][] puzzle：挖空后的题目（实现中主要用于初始化，逻辑上以 playerBoard + fixed 为准）。
  • int[][] playerBoard：当前玩家可见/可编辑盘面（0 表示空）。
  • boolean[][] fixed：true 表示题目预填，不可 placeNumber 覆盖（除规则外）或 clearCell。
  • boolean gameStarted / gameOver；int difficulty；Random random。
题目生成流程
  1. fillBoard(board, row, col)：按行列顺序递归回溯，每格从打乱顺序的 1–9 中选取，调用 isValid 检查行/列/宫约束，填满即得 solution。
  2. 复制 solution 到 puzzle 与 playerBoard。
  3. 将 0..80 位置打乱，依难度删除对应数量格子（置 0）。
  4. 根据 puzzle 设置 fixed[r][c] = (puzzle[r][c] != 0)。

API Actions — 参数与输出（与源码一致）

所有请求参数为 JSON 字符串 paramsJson；空或空白视为 {}。optInt 未传字段时使用默认值（如 row 默认 -1 会触发越界 error）。

startGame
输入参数difficulty：int，可选；未传时 optInt(..., 2)；随后 Math.max(1, Math.min(3, diff)) 夹紧到 1–3。
成功 output由 buildState() 生成，字段见下表「公共 state 字段」。另含整局 formatDisplay() 文本于顶层 _displayText。
placeNumber
输入参数row，col（optInt(..., -1)），number（optInt(..., 0)）
错误条件未 startGame；row/col 非 0–8；number 非 1–9；fixed[row][col] 为 true。
成功 output公共字段 + placedCorrect（number == solution[row][col]）。若放错数字，附 message：「数字不正确，但已放置」。若全盘填满且与 solution 一致：gameOver=true，completed=true，message 祝贺文案。
clearCell

params：row，col。仅当非 fixed 时将 playerBoard 置 0。

checkBoard

output 附加字段：complete（isBoardComplete()），correct（isBoardCorrect()），errors（与 solution 不一致的非 0 格数量），emptyCells。若 complete && correct：gameOver=true 与祝贺 message。

getHint

候选格：满足 playerBoard[r][c]==0 || playerBoard[r][c]!=solution[r][c]。随机选一格写入 solution 值。返回 hintRow，hintCol，hintValue。若此后全盘正确则 completed 与 gameOver。

getState / restart

getState：需 gameStarted。

restart：若 difficulty==0 置 2；若 random==null 则 new Random()；generatePuzzle() 全新题目。
