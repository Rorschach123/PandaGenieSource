模块概述

通过 ContentResolver 与 ContactsContract 读取系统通讯录：搜索、分页列表、详情、计数、vCard 3.0（.vcf）全量导出，以及按规范化显示名与规范化电话号码（仅数字）检测重复分组。

权限： READ_CONTACTS。Capability： contacts。

架构与数据流

invoke → searchContacts: CONTENT_FILTER_URI + Phone LIKE → getContactDetail: Contacts + Data 子表 loadPhones/Emails/Addresses/Organizations → listContacts: IN_VISIBLE_GROUP=1, COLLATE LOCALIZED ASC + offset/limit → exportContacts: 游标遍历 → writeVcard per contact → UTF-8 文件 → findDuplicates: 全量姓名 Map + 全量电话 Map ← ok(output JSON string, _displayText)

API Actions — 输入 / 输出

searchContacts
参数说明
query关键词；空则返回 total:0
limit默认 20；范围 clamp 到 [1,500]

流程：① Contacts.CONTENT_FILTER_URI + encode(query)，过滤 IN_VISIBLE_GROUP=1，LinkedHashSet 保序去重。② 若 query 含数字且未满 limit，Phone.NUMBER LIKE %digits% 补充。

output： contacts[]：{ id, name, primaryPhone }，total。

getContactDetail

必填 contactId（Contacts._ID）。

成功 output： contactId, displayName, lookupKey, phones[], emails[], addresses[], organizations[]。

子元素字段见 loadPhones（number/type/label/isPrimary）等。

listContacts

limit 默认 50，最大 2000；offset 默认 0。

output： contacts 当前页，total 为游标总条数（非仅当前页）。

getContactCount

与列表相同可见性：IN_VISIBLE_GROUP=1。output： {"count":N}。

exportContacts

outputPath 默认 /sdcard/PandaGenie/output/contacts_export.vcf。成功：path, exported；失败：内层 error 字符串 → 外层 success:false。

findDuplicates

duplicateNameGroups： normalizeName（trim + lower）相同且人数>1。

duplicatePhoneGroups： digitsOnly(number) 长度 ≥5，同一规范化号码对应多个 contactId。

每组：{ normalizedName|normalizedPhone, contactIds, count }。

VCard 3.0 导出细节

writeVcard 写入 BEGIN:VCARD / VERSION:3.0 / FN:，随后 TEL、EMAIL、ADR;TYPE=HOME、ORG/TITLE，END:VCARD。换行 \r\n，值经 escapeVcardValue / escapeVcardParam。

电话类型映射：phoneTypeToVcard；邮箱：emailTypeToVcard。
