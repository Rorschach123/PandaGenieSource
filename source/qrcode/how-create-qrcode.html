QR 引擎内嵌 Nayuki QR Code generator（Java），MIT License

本模块在宿主 WebView 中加载 index.html，通过 NativeBridge 调用 Java 插件；解码路径采用浏览器端 jsQR，与 Java 端生成链路分离，兼顾性能与内存安全。

三层架构 · 3-Layer Architecture

  1. Presentation（HTML + JavaScript）：index.html，三标签页 UI；扫描页使用 jsQR 在客户端解码。
  2. Bridge（NativeBridge）：invoke(action, paramsJson) / pickImage() 等与宿主交互。
  3. Plugin（Java）：QrcodePlugin 使用 Nayuki 的 QrCode 生成矩阵 → Bitmap → PNG 文件；可选返回 Base64。
解码策略：Generation 在 Java 侧完成（Nayuki → Bitmap → PNG）。Decoding 在 WebView 内用 jsQR 完成；Java 的 decodeQR 仅校验路径并读取 inJustDecodeBounds 元数据，并返回 hint: open_scan_tab 引导 UI。
  • Gallery：Java 列举外部存储 PandaGenie/qrcode/ 下 .png，按 修改时间降序。

QrcodePlugin.java

约 370 行。实现 ModulePlugin，统一入口 invoke(Context, String action, String paramsJson)。

常量 · Constants

名称说明
SIZE_MIN64 — 生成位图边长下限（像素）
SIZE_MAX2048 — 上限，防止 OOM
SIZE_DEFAULT512 — 未指定 size 时的默认边长
BASE64_MAX_SIDE512 — 边长 ≤ 此值时默认附带 Base64，避免大 JSON

invoke() — 三种 action

action职责
generateQR生成 QR、写 PNG、返回路径与元数据；可选 Base64
decodeQR校验图片路径、仅读 bounds、返回元数据 + hint
listGenerated列出输出目录 PNG，按修改时间降序
generateQR() 细节
  • 使用 QrCode.encodeText(text, QrCode.Ecc.MEDIUM) — MEDIUM ECC。
  • renderQrBitmap()：4 模块静区（quiet zone），按单元格在 Canvas 上绘制。
  • 输出目录：Environment.getExternalStorageDirectory()/PandaGenie/qrcode/，文件名 qr_<timestamp>.png。
  • 若 includeBase64 == true 或 size <= BASE64_MAX_SIDE，附加 PNG 的 Base64。
decodeQR() 细节
  • 参数：imagePath（必填）。
  • BitmapFactory.Options.inJustDecodeBounds = true — 只读尺寸与 MIME，不解码全图。
  • 返回字段含 width、height、fileSizeBytes、mimeType、decoded: null、hint: "open_scan_tab"。
listGenerated() 与展示格式化
  • 列出 .png，排序：lastModified 降序。
  • formatGenerateDisplay / formatDecodeDisplay / formatListDisplay：生成 _displayText 供宿主 UI 展示。
  • 辅助方法：recycleQuietly、emptyJson、ok、error。

Nayuki QR 嵌入库 · Embedded Library

包路径：ai.rorsch.moduleplugins.qrcode.qr。以下为各类职责摘要（行数为仓库内近似统计）。

QrCode.java（约 815 行）
  • QR 码生成的核心类；版本 MIN_VERSION = 1 … MAX_VERSION = 40。
  • Enum Ecc： LOW、MEDIUM、QUARTILE、HIGH（含 ordinal、formatBits、ECC codewords 等表）。
  • 工厂方法： encodeText、encodeBinary、encodeSegments(segs, ecl, minVer, maxVer, mask, boostEcl) 等。
  • 实例字段： version、size、errorCorrectionLevel、mask、modules[][]、isFunction[][]。
  • 关键方法： getModule(x,y)、drawFunctionPatterns、drawFormatBits、drawVersion、drawCodewords、applyMask、getPenaltyScore。
  • 静态辅助： getBit、addEccAndInterleave、getNumRawDataModules、getNumDataCodewords 等。
QrSegment.java（约 317 行）
  • 数据段：makeBytes、makeNumeric、makeAlphanumeric、makeSegments、makeEci。
  • 校验：isNumeric、isAlphanumeric。
  • Enum Mode： NUMERIC、ALPHANUMERIC、BYTE、KANJI、ECI — modeBits、numCharCountBits。
  • ALPHANUMERIC_CHARSET 常量。
BitBuffer.java（约 129 行）

基于 BitSet 的可追加位序列。方法：bitLength、getBit、appendBits、appendData、clone。

DataTooLongException.java（约 57 行）

继承 IllegalArgumentException，载荷超出 QR 容量时抛出。

index.html UI

  • 三标签页： Generate（生成）、Scan（扫描）、Gallery（图库）。
  • Generate： 文本/链接输入、边长下拉（256–1024 等）、调用 generateQR 且 includeBase64: true 用于预览。
  • Scan： NativeBridge.pickImage() 选图；onImagePicked(path) 加载 file://；可选调用 decodeQR 做校验；jsQR 在 canvas 上解码（含缩放与 inversionAttempts: 'attemptBoth'）。
  • Gallery： listGenerated 拉取列表并网格展示缩略图。
  • i18n： zh / en 字典，NativeBridge.getLanguage() 或浏览器语言回退。

脚本依赖：jsQR（CDN：jsqr@1.4.0）；样式可引用同目录 common.css（Material 变量）。

API 参考 · Plugin Actions（JSON）

所有调用经宿主封装：成功返回外层 JSON，含 success: true、output（字符串化的业务 JSON），以及可选 _displayText。

1. generateQR

请求参数（params JSON）

字段类型说明
textstring必填，编码内容（trim 后非空）
sizeint可选，像素边长；钳制在 [SIZE_MIN, SIZE_MAX]，默认 SIZE_DEFAULT
includeBase64bool可选；true 时强制附带 PNG Base64（大图慎用）

output 字段（成功，业务 JSON）

字段说明
path绝对路径（Absolute path）到已保存 PNG
text原始文本
requestedSize请求尺寸（钳制后）
imageSizeBitmap 实际边长（可能因整除单元格略小于请求）
qrVersionNayuki 版本号（1–40）
modules逻辑模块边长（module count）
base64可选，PNG 的 Base64（NO_WRAP）

典型错误： 空文本、目录不可写、DataTooLongException → “text too long for QR code”、PNG 压缩失败等。

2. decodeQR

请求参数

字段说明
imagePath必填，本地图片绝对路径

output 字段

字段说明
imagePath / width / height路径与像素尺寸
fileSizeBytes文件大小
mimeType来自 bounds，可能为 null
decodednull（正文解码在 Scan 页由 jsQR 完成）
hint"open_scan_tab" — 建议打开扫描标签

3. listGenerated

请求参数： 无（空对象 {} 即可）。

output 字段

字段说明
directory扫描目录绝对路径
files数组：每项含 name、path、sizeBytes、modified
count文件数量

QR 生成流程 · Generation Flow

  1. Web UI 收集 text、size、includeBase64。
  2. NativeBridge.invoke('generateQR', params) → QrcodePlugin。
  3. QrCode.encodeText(..., Ecc.MEDIUM) 得到模块矩阵。
  4. renderQrBitmap：4 模块白边 + 逐格绘制 → ARGB_8888 Bitmap。
  5. 写入 PandaGenie/qrcode/qr_*.png；按需附加 Base64。
  6. 返回 JSON；前端可用 data:image/png;base64,... 预览。
