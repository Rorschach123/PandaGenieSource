模块概述 Module overview

签名校验（signature_checker） 用于：读取当前宿主 APK 的签名证书（PackageManager.GET_SIGNATURES）并输出 SHA-256 fingerprint、Subject、Issuer； 对外部存储目录 <ExternalStorage>/PandaGenie/modules/*.mod 的模块包进行 JAR 校验（JarFile(modFile, true)）， 比对条目证书与 assets 中内置 官方 PEM 证书（module_signing/official_cert.pem）是否一致，从而区分 isOfficial 与开发者证书指纹。

Capabilities： file_read。

架构 Architecture

  • APK： 取 signatures[0]，CertificateFactory 解析为 X509Certificate，sha256Fingerprint(getEncoded())。
  • 模块 .mod： 作为 JAR/ZIP；逐条读取非 META-INF/ 条目并读完流以触发 JAR 摘要验证；每条目须带 Certificates，否则失败。
  • manifest.json： 自模块包内条目读取，解析 id、name、version、developer；BOM 会剥离。
  • 官方证书： loadTrustedCert 从 assets 读 PEM，Base64 解码后实例化 X509Certificate；失败则模块侧校验标记为未验证并带 error。

输出字段说明（模块校验成功时示例）

成功校验且 verified: true 时常见字段：file, id, name, version, developer, verified, signerCount, isOfficial, officialFingerprint, officialSubject, devFingerprint, devSubject。失败时 verified: false 与 error 说明原因（未签名条目、篡改 SecurityException、无证书等）。

// 外层统一
{ "success": true, "output": "<JSON string>", "_displayText": "..." }

数据流 Data flow

ExternalStorage/PandaGenie/modules/*.mod → JarFile 校验 + manifest.json → 对比 assets official_cert.pem 指纹 → JSONObject 结果 → ok(output, displayText)
