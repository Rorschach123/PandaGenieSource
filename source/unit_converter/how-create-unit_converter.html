模块概述

在本地完成 length、weight、temperature、area、volume、speed、time、data 八类单位互转。非温度类通过「乘到基准量（base）再除以目标单位因子」实现；温度先统一为 摄氏 Celsius 再转到目标温标。

常量：MONTH_SECONDS = 30×86400（月近似），YEAR_SECONDS = 365×86400（年近似）。

换算模型

  • convert(category, value, from, to)：若 temperature，则 toCelsius(from,value) → fromCelsius(to, c)；否则 base = value * factorToBase(from)，result = base / factorToBase(to)。
  • normalizeCategory：小写 trim，且必须在 CATEGORY_TITLES 键集合中。
  • normalizeUnitToken(category, raw)：去空格、²→2、³→3，再按类别映射别名（如 kmh→km/h，°c→C）。
  • formatNumber：整数无小数；极小/极大用科学计数；否则 %.10g。

类 UnitConverterPlugin — 主要方法

方法说明
invokeconvert|listUnits|listCategories
listUnitKeys按类别返回有序键列表（LinkedHashMap 插入顺序或显式顺序）
normalizeCategory / normalizeUnitToken类别与单位规范化
convert / toBase / fromBase核心换算
toCelsius / fromCelsius温标转换公式
require(Map,String)查表，缺失抛 Unknown unit
displayUnitLabel / categoryTitle / categoryListTitle展示用标签与标题
ok / error标准响应；convert 的 _displayText 含 emoji 标题与等式一行

API Actions — 输入 / 输出

convert

params：value（必填，数字），fromUnit，toUnit，category

错误：Invalid value（非数字）；Unknown or missing unit；未知类别等。

成功 output：result（double），resultText（格式化字符串），fromUnit，toUnit（规范化键），category。

_displayText 示例结构：📐 Unit Conversion\n{value} {prettyFrom} = {resultText} {prettyTo}\nCategory: {Length|…}

listUnits

params：category

成功 output：category（规范化），units（JSON 数组字符串元素顺序即 UI 顺序）

失败：Unknown category

listCategories

params：无

output：categories 数组，顺序为静态 CATEGORY_TITLES 插入顺序：length, weight, temperature, area, volume, speed, time, data。

数据流

convert:
  optDouble("value") → NaN? → error
  normalizeCategory(category) → ""
  normalizeUnitToken(category, from/to) → ""
  convert() → formatNumber(result)
  JSONObject + _displayText

listUnits:
  listUnitKeys → JSONArray

listCategories:
  CATEGORY_TITLES.keySet() → JSONArray
