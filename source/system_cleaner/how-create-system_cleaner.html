模块概述

在外部存储（Environment.getExternalStorageDirectory()）上按类别扫描（scan）可清理文件体积、或删除（clean）以释放空间；另提供 StatFs 存储概览与大文件枚举。实现 ModulePlugin，由宿主 ModuleRuntime 反射调用；需由宿主申请 存储读写权限（READ/WRITE_EXTERNAL_STORAGE 或分区存储适配）。

类别（categories） 逗号分隔、小写：temp、cache、apk、thumbs、empty_dirs、logs。scanJunk 在参数为空时默认扫描全部六类；cleanJunk 必须显式指定至少一类。

架构与类别语义

Category Key扫描范围（摘要）清理行为
temp<ext>/PandaGenie/temp 递归；全盘 *.tmp（排除已统计目录内重复）删除 PandaGenie temp 目录内文件 + 其它 .tmp
cacheAndroid/data/*/cache 各包外部缓存递归清空各 cache 目录内容
apk外存根目录 *.apk；Downloads 与 Download 树内 APK同上范围删除 APK 文件
thumbs深度优先查找名为 .thumbnails 的目录并统计其内文件删除每个 .thumbnails 整树
empty_dirs统计「叶子空目录」数量（非根），用于估算空文件夹规模多轮（最多 64 轮）扫除空目录，每轮自底向上删除空文件夹
logs全盘递归匹配 *.log删除匹配日志文件

注意：empty_dirs 的「大小」在扫描阶段仅计目录个数（CategoryStat.addEmptyDir），字节为 0。

类、内部类型与主要方法

公开入口

invoke(Context context, String action, String paramsJson)：分发 scanJunk、cleanJunk、getStorageInfo、findLargeFiles；各动作结果经 ok(outputString, displayText) 包装。

内部静态类

  • CategoryStat：count，sizeBytes；addFile(long)，addEmptyDir()；toJson() 输出 count、sizeBytes、sizeFormatted。
  • CleanResult：files，dirs，bytes；toJson() 含 filesDeleted、dirsDeleted、freedBytes、freedFormatted。
  • FilePredicate：函数式接口，用于 accumulateMatchingFiles / deleteMatchingFiles。

StatFs 与格式化

statFsToJson(File path)：兼容 API 17+ 长整型块大小/块数；字段含 totalBytes、availableBytes、usedBytes、usedPercent、totalFormatted 等（GB/TB 英文单位）。

getStorageInfo()：internal ← Environment.getDataDirectory()；externalPrimary ← 外存目录并附加 state（getExternalStorageState()）。

文件遍历辅助（节选）
  • accumulateFilesRecursive、accumulateMatchingFiles
  • collectLargeFiles：非递归或递归收集 ≥ 阈值文件
  • isUnderDirectory：规范路径前缀判断，避免重复统计

API Actions — 输入 / 输出

scanJunk

params：categories（可选，逗号分隔；空则六类全扫）

成功 output（JSON 对象，字符串在顶层 output 内）：

  • categories：各 key 对应 CategoryStat.toJson()
  • totalSizeBytes，totalSizeFormatted

外存不可用：output 内含 {"error":"External storage not available"}（仍可能 success:true，展示层识别 error 键）。

cleanJunk

params：categories（必填非空）

output：deletedFiles，deletedDirs，deletedItems，freedBytes，freedFormatted，byCategory（每类 CleanResult）。

getStorageInfo

params：无

output：internal，externalPrimary（可能为 null JSON 字面量）。

findLargeFiles

params：

  • minSizeMB：默认 50.0
  • path：默认外存根或 /sdcard
  • recursive：默认 true
  • limit：默认 100（manifest 未列出的扩展参数）

output：directory，minSizeMB，found，showing，files[]（每项含 path、name、size、sizeFormatted）。按文件大小降序截取前 limit 条。

数据流与安全说明

User / Agent
    │  action + paramsJson
    ▼
invoke() → switch(action)
    ├─ scanJunk  → scan* 系列（只读 Stat）→ categories JSON
    ├─ cleanJunk → clean* / delete*（写文件系统）→ freed 统计
    ├─ getStorageInfo → StatFs → internal / external JSON
    └─ findLargeFiles → collectLargeFiles → sort → top N

响应封装: ok(businessJsonString, _displayText Markdown 风格表格)

风险提示：cleanJunk 会删除真实文件与空目录；clean cache 影响各应用外部缓存。生产环境应在 UI 侧二次确认并遵守 Scoped Storage 策略。
