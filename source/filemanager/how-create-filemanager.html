本模块在宿主中通过 ModulePlugin 契约暴露 13 种 action，底层由 JNI 调用 filemgr::FileManager 完成 POSIX 风格文件操作；所有路径在 native 层经 SandboxGuard 校验。

四层架构 · 4-Layer Architecture

1. HTML / Chat UI任务或对话层发起 JSON 调用（action + params），展示 _displayText 摘要。
2. Java Plugin（FileManagerPlugin）解析参数、resolveStoragePath、调用 FileManagerLib、格式化人类可读输出。
3. JNI Bridge（filemanager_jni.cpp）jstring ↔ std::string，JSON 序列化，异常映射为 RuntimeException；静态方法对接 nativeConfigureSandbox / nativeClearSandbox。
4. C++ Native（filemanager.cpp + sandbox_guard.cpp）POSIX I/O（dirent、stat、rename 等），每项操作前 SandboxGuard::checkRead/Write。

类与源码说明 · Class Documentation

FileManagerPlugin.java（约 392 行）

职责：实现 ModulePlugin，作为模块唯一 Java 入口。

  • 常量：DISPLAY_LIST_MAX = 20（列表展示行数上限）、READ_TEXT_DISPLAY_MAX = 2000（读文本展示截断）、SDF 日期格式 yyyy-MM-dd HH:mm:ss
  • 字段：private final FileManagerLib lib
  • invoke()：13 个 action：listDirectory、createDirectory、deleteFile、deleteDirectory、copyFile、moveFile、renameFile、getFileInfo、searchFiles、readTextFile、writeTextFile、fileExists、getFileSize
  • handleCopyOrMove：校验源存在；若 dst 为目录则拼接源文件名；mkdirs 创建父目录；移动/复制失败时附带不可写/不可读提示
  • resolveStoragePath：/sdcard 前缀映射到 Environment.getExternalStorageDirectory()
  • Display formatters：formatListDirectoryDisplay、formatRenameDisplay、formatCopyDisplay、formatMoveDisplay、formatGetFileInfoDisplay、formatSearchFilesDisplay、formatReadTextFileDisplay
  • Helpers：parseBoolean、emptyJson、ok、boolResult、error、formatSizeBytes

  • 头文件：dirent.h、sys/stat.h、unistd.h 等；日志 android/log.h
  • Sandbox：每个公开操作前调用 sandbox::SandboxGuard::checkRead 或 checkWrite（移动/重命名涉及多路径）

API 参考 · 13 Actions

统一通过 ModulePlugin.invoke(context, action, paramsJson) 调用。成功响应通常含 success: true 与 output（字符串：JSON 或标量）；部分含 _displayText。失败为 success: false 与 error。

action 参数（params JSON） output 含义
listDirectory path：目录路径 JSON 数组，元素为 FileInfo（name, fullPath, isDirectory, size, lastModified）
createDirectory path "true" / "false"
deleteFile path 布尔字符串
deleteDirectory path、recursive（默认 false） 布尔字符串；recursive 为 true 时递归删除
copyFile src / source / path；dst / destination / target "true"；目标为目录时自动使用源文件名
moveFile 同 copyFile 同上
renameFile oldPath、newPath 布尔字符串
getFileInfo path 单个 FileInfo JSON 对象字符串
searchFiles dir（默认 "/sdcard"）、pattern（默认 "*"）、recursive（默认 true，支持 Boolean 或字符串） 匹配文件全路径 JSON 数组字符串；native 最多 5000 条
readTextFile path 文件全文；展示层截断至 READ_TEXT_DISPLAY_MAX
writeTextFile path、content 布尔字符串（覆盖写入）
fileExists path "true" 或 "false"
getFileSize path 字节数字符串；native 失败为 -1

Sandbox 安全模型 · Sandbox Security

设计目标：在授予存储权限的前提下，仍由宿主通过 nativeConfigureSandbox 限定 native 层可读/可写路径前缀，降低模块越权访问风险。

  • 读操作（列举、读文件、搜索、exists、getSize、getFileInfo 等）在对应路径上调用 SandboxGuard::checkRead。
  • 写操作（创建/删文件、写文本、复制目标端等）调用 SandboxGuard::checkWrite；move 同时校验源读、源写、目标写。
  • 配置入口：Java FileManagerLib.nativeConfigureSandbox(readPaths, writePaths) → JNI → sandbox_jni::configureSandbox；nativeClearSandbox 清除策略。
  • 与 Manifest：MANAGE_EXTERNAL_STORAGE 解决系统级访问；Sandbox 为应用内二次约束，二者互补。

典型调用链（示例：listDirectory）

  1. UI 发送 action=listDirectory，params.path 经 Java 规范化。
  2. nativeListDirectory → FileManager::listDirectory → checkRead(path) → opendir/readdir/stat。
  3. JNI 将 vector<FileInfo> 拼为 JSON 数组返回；Plugin 再生成 _displayText。
