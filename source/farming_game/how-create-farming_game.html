模块概览

本模块为 PandaGenie 的种菜模拟娱乐插件：玩家在 6 块地块（plots）上种植 6 种作物（crop types），通过 浇水 / 施肥 / 除草影响生长速度，成熟后 收获并记录 品质（quality）与产量。startGame 会覆盖唯一存档；enterGame 加载存档并在无档时自动新开。

manifest 要点：capabilities 含 file_read、file_write；需 MANAGE_EXTERNAL_STORAGE 以写入存档。pluginClass 为 ai.rorsch.moduleplugins.farming_game.FarmingPlugin。

架构与存储

  • 入口：invoke(Context, action, paramsJson) 解析 JSON，按 action 分发。
  • 状态：内存中 JSONObject gameState，字段含 startTime、lastUpdateTime、plots（JSONArray）、harvestHistory、totalHarvests。
  • 存档：外部存储 Environment.getExternalStorageDirectory() 下 PandaGenie/data/farming_game/save.json，UTF-8，JSONObject.toString(2) 美化输出。
  • 作物表 CROP_DATA：每行 seedType, 中文名, emoji, growStages, baseGrowTime(分钟), harvestYield。
  • 生长：updateGrowth() 根据 lastUpdateTime 与当前时间差（分钟）更新每块地的 accumulatedGrowth、currentStage、mature，并衰减 waterLevel/fertilizerLevel、增长 weedLevel。

类 FarmingPlugin 与方法

单类实现，常量：PLOT_COUNT=6，SAVE_DIR、SAVE_FILE，GROWTH_EMOJIS 用于展示。

API Actions（输入 / 输出）

成功时外层：{"success":true,"output":"...JSON字符串...","_displayText":"..."}。

startGame / enterGame

params：无。

output 内嵌 JSON（解析后）：gameLoaded、totalHarvests、plots[]（含 index、empty、作物时 seedType、cropName、stage、水肥草等）。

plant
参数类型说明
plotIndexint0–5
seedTypestringtomato/carrot/cabbage/corn/watermelon/strawberry

输出：更新后的展示状态；错误时顶层 success:false。

water / fertilize / weed / harvest

params：plotIndex（0–5）。harvest 成功时 output 内可能含 message 描述产量与品质。

getState / getHarvestHistory / saveGame

getHarvestHistory output：harvestHistory（JSONArray 记录 cropName、cropEmoji、yield、quality、harvestTime）、totalHarvests、_historyDisplay。

saveGame：output 可含 message；_displayText 为简短确认。

数据流 Data Flow

User/LLM → invoke(action, paramsJson) → JSONObject params → 各 action 修改 gameState → updateGrowth()（时间推进） → saveToFile() → ExternalStorage/.../save.json → ok(buildDisplayState(), formatDisplay()) → { success, output, _displayText }

首次操作前须 startGame 或 enterGame，否则 ensureLoaded 抛错。定时任务场景下，依赖系统多次调用间的时间差驱动 updateGrowth。
