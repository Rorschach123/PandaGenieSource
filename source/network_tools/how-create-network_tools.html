模块概述 Module overview

网络工具（network_tools） 为 PandaGenie 提供本地网络诊断与信息查询能力：通过系统 ping 子进程、 InetAddress DNS 解析、网卡枚举、HTTPS 公网 IP 查询、ConnectivityManager 与 WifiManager 等 Android API， 将结果序列化为 JSON 字符串，并由宿主 ModuleRuntime 经 ModulePlugin#invoke 返回给前端或助手。

Capabilities： manifest 声明 network。Permissions： ACCESS_NETWORK_STATE、ACCESS_WIFI_STATE（用于连通性与 Wi‑Fi 信息）。

架构 Architecture

  • 入口： 实现接口 ModulePlugin，唯一公开实例方法 invoke(Context, String action, String paramsJson)。
  • 分发： switch (action) 映射到 ping、dnsLookup、getLocalIp、getPublicIp、checkConnectivity、getNetworkInfo。
  • 安全： ping 的 host 使用正则 SAFE_HOST 校验，防止 command injection。
  • 子进程： ping 使用独立线程 drainStream 读取 stdout/stderr，避免缓冲区死锁；API 26+ 使用 Process#waitFor(timeout)，低版本使用轮询 waitForProcess。
  • 展示： 成功时通过私有 format*Display 生成 Markdown 风格表格到字段 _displayText。

API Actions（输入 / 输出）

所有请求参数置于 paramsJson 解析后的 JSONObject 中。成功时外层见「响应封装」。

ping

输入： host（string，必填）— 主机名或 IP，须匹配 SAFE_HOST。

output 内嵌 JSON 字段： host, command, stdout, stderr, exitCode, timedOut。

dnsLookup

输入： domain（必填）。

output： domain, addresses（字符串数组）。

getLocalIp

输入： 无。

output： ipv4[], ipv6[]。

getPublicIp

输入： 无。需网络。

output： httpStatus, body, ip（HTTP 2xx 且 JSON 解析成功时）。

checkConnectivity

输入： 无（使用 Context）。

output： hasConnectivityService, activeNetwork, hasInternetCapability, validated, metered, transports（如 wifi/cellular/ethernet/vpn 等）。

getNetworkInfo

输入： 无。

output： networkType, connected, wifi 相关字段（ssid/bssid/link speed 等，依系统策略可能为空）。

响应封装 Response envelope

成功：

{
  "success": true,
  "output": "<业务 JSON 字符串，需二次 parse>",
  "_displayText": "<可选，Markdown 风格摘要>"
}

失败（invoke 捕获异常或未知 action）：

{ "success": false, "error": "<message>" }

数据流 Data flow

Host App (ModuleRuntime) → 反射加载 NetworkToolsPlugin → invoke(context, action, paramsJson) → 业务方法（ping / dns / …） → ok(outputString, displayText) 或 error(msg) → WebView / Chat UI 解析 JSON，展示 output 与 _displayText
