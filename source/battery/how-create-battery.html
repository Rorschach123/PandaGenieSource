模块概述

本模块（battery）通过 Android 系统粘性广播 Intent.ACTION_BATTERY_CHANGED 读取电量、充放电状态、插电类型、温度、电压、健康状态与电池技术字符串。不依赖厂商私有 API，异常统一转为 success: false JSON。

核心类： android.os.BatteryManager（EXTRA_* 常量）、IntentFilter(ACTION_BATTERY_CHANGED)。Android 13+（TIRAMISU）注册粘性接收器时使用 Context.RECEIVER_NOT_EXPORTED。

架构与数据流

H5 index.html（说明页） → NativeBridge.invoke(action, "{}") → BatteryPlugin.invoke() → getBatteryIntent(): registerReceiver(null, IntentFilter(ACTION_BATTERY_CHANGED)) → 从 Intent extras 读取 LEVEL/SCALE/STATUS/PLUGGED/TEMPERATURE/VOLTAGE/HEALTH/TECHNOLOGY ← JSON { success, output, _displayText }

电量百分比计算：pct = round(100 * level / scale)，当 level < 0 或 scale <= 0 时 levelPercent 为 -1。

API Actions — 输入 / 输出

getBatteryStatus

输入： 无（params 可为 {}）。

output 字段（JSON 字符串）：

字段说明
levelPercent0–100 或 -1（未知）
chargingboolean，STATUS 为 CHARGING 或 FULL 时为 true
statuscharging|discharging|full|not_charging|unknown
pluggedTypenone|usb|ac|wireless|unknown
pluggedRaw原始 int 位标志
temperatureC摄氏度 double，或 JSON null
voltageMilliVolts毫伏 int 或 null
healthGood/Cold/Dead/Over voltage/Overheat/Unspecified failure/Unknown
healthRaw整型原始健康常量
technology如 Li-ion 字符串，可能为空
getBatteryHealth

在 buildStatusJson 基础上增加健康报告字段：

  • healthStatus — 与 status 中 health 字符串一致
  • temperatureWarning — 中文温度区间提示（≥45°C 偏高、40–45 略高等）
  • cycleEstimate / cycleEstimate_en — 说明标准广播通常不提供真实循环次数
  • currentLevelPercent
  • overallAssessment — 综合中文评估
getPowerSummary

合并状态与健康摘要，含中文一行 summaryLine（电量、状态中文、插电中文、温度、健康）。

主要字段： levelPercent, status, charging, pluggedType, temperatureC, voltageMilliVolts, health, technology, summaryLine, healthStatus, temperatureWarning。

内部 try/catch：异常转为 error(异常消息)。成功时均带 _displayText（Markdown 表格）。

getBatteryIntent(Context)

registerReceiver(null, filter) 获取最近一次 ACTION_BATTERY_CHANGED 粘性 Intent；不可用时后续构建抛出 IllegalStateException。

buildStatusJson / buildHealthReportJson / buildPowerSummaryJson

三层组装逻辑；buildPowerSummaryJson 调用前两者的结果字段。
