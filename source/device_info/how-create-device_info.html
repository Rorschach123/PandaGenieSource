模块概述

聚合本机静态 Build 信息、/proc/cpuinfo（截断）、ActivityManager.MemoryInfo、StatFs + Environment 存储路径、WindowManager + DisplayMetrics 与刷新率，以及一次性 getSystemSummary 嵌套树。仅使用公开 Android API 与只读 proc 节点。

架构与数据流

invoke(action, params) → getDeviceInfo() → Build.* → getCpuInfo() → Runtime.availableProcessors, os.arch, SUPPORTED_ABIS, /proc/cpuinfo → getMemoryInfo(ctx) → ActivityManager.getMemoryInfo → getStorageInfo() → statFsToJson(getDataDirectory), statFsToJson(getExternalStorageDirectory) → getDisplayInfo(ctx) → Display.getRealSize (API 17+), DisplayMetrics, refresh rate → getSystemSummary() → 合并上述五段 JSON ← ok(output, _displayText)

API Actions — 输入 / 输出

getDeviceInfo

无参数。字段： brand, model, manufacturer, androidVersion（Build.VERSION.RELEASE）, sdkInt, buildNumber（Build.DISPLAY）, buildIncremental, fingerprint。

getCpuInfo

processorCount = Runtime.getRuntime().availableProcessors()。

osArch = System.getProperty("os.arch")。

supportedAbis = JSONArray(Build.SUPPORTED_ABIS)。

cpuinfo：读取 /proc/cpuinfo 最多 CPUINFO_MAX_CHARS=65536 字符；统计以 processor 开头的行数为 processorEntriesInCpuinfo。IO 失败时设置 cpuinfoError。

getMemoryInfo

若 ACTIVITY_SERVICE 不可用，返回内含 error 的 JSON（仍包装 success，由 UI 展示）。

否则：totalBytes, availableBytes, usedBytes=total−avail，usedPercent 保留两位小数，lowMemory，API 16+ thresholdBytes。

getStorageInfo

根对象含 internal、externalPrimary，路径不存在时为 JSON null。

子对象（statFsToJson）：path, totalBytes, availableBytes, usedBytes, usedPercent。外部卷额外 state = Environment.getExternalStorageState()。

getDisplayInfo

getMetrics 取 density；宽高优先 Display.getRealSize(Point)（API 17+）以适配全面屏。

字段： widthPx, heightPx, density, densityDpi, refreshRateHz（API R+ 用 display.getRefreshRate()；M–Q 用 getMode().getRefreshRate()；更旧用 deprecated getRefreshRate()）。

getSystemSummary

结构： { "device": {...}, "cpu": {...}, "memory": {...}, "storage": {...}, "display": {...} }，各子对象为对应私有方法返回 JSON 的解析结果。
