模块概述 Module overview

提醒助手（reminder） 通过 Android CalendarContract 读写系统日历（事件 CRUD、实例查询、生日每年重复、临近日程）； 通过 AlarmClock.ACTION_SET_ALARM 与 AlarmClock.ACTION_SET_TIMER 调起系统闹钟/倒计时界面； 通过 Intent.ACTION_VIEW 打开系统日历到指定时间戳。

Capabilities： calendar。Permissions： READ_CALENDAR、WRITE_CALENDAR。

架构 Architecture

  • 时间格式：SDF_DATETIME = yyyy-MM-dd HH:mm；日期查询 SDF_DATE = yyyy-MM-dd；列表展示 SDF_DISPLAY 含秒。
  • 事件实例查询使用 CalendarContract.Instances 内容 URI（区间由 ContentUris.appendId 限定）。
  • createEvent：重复事件时写入 RRULE，移除 DTEND 并设置 DURATION（PT1H），符合 Android 日历契约。
  • getDefaultCalendarId：优先 IS_PRIMARY，否则第一个 VISIBLE=1 日历。

错误语义与 errJson

重要： createEvent、listEvents 等失败路径返回字符串 errJson("...")， 该字符串仍作为 success: true 的 output 字段内容；客户端应 parse output 后检查 error 键。 仅当 invoke 遇到未知 action 时才会 success: false（error()）。

updateEvent、deleteEvent 等同样可能内嵌业务失败信息于 output JSON。

数据流 Data flow

invoke → CalendarContract ContentResolver / AlarmClock Intent → output JSON string → ok(output) 或 ok(output, _displayText) → UI 解析 success + output（注意内层 error）
