模块概述 Module overview

笔记（notes） 模块在固定目录 /sdcard/PandaGenie/data/notes/ 下以 每篇笔记一个 JSON 文件（{id}.json）持久化数据。 支持创建、列表（含可选关键词过滤）、按 ID 读取全文、按字段更新、删除、搜索（关键词必填）以及导出为纯文本 .txt。

Capabilities： file_read、file_write。 Permissions： MANAGE_EXTERNAL_STORAGE（manifest 描述用于本地存储读写；实际以宿主权限为准）。

文档说明：用户场景下亦称「便签/笔记」；本页以源码与 manifest.json 为准。

数据模型与存储 Data model

单篇笔记 JSON 字段（由 createNote / updateNote 维护）：

字段说明
id唯一 ID：毫秒时间戳 + 随机后缀，冲突时重试最多 256 次。
title标题，创建时必填非空（trim 后）。
content正文，可为空串。
createdAt创建时间，格式 yyyy-MM-dd HH:mm:ss（默认 Locale）。
updatedAt创建或每次更新时刷新。

列表排序： loadAllNotes() 按 updatedAt 字符串降序（字典序等价于时间先后，因格式固定）。

预览： PREVIEW_MAX = 120，超出截断加 ...。

架构 Architecture

  • ModulePlugin 单类实现，无其它 Java 类文件。
  • invoke 捕获 IllegalArgumentException 转为 error(message)；其它异常继续向上抛出（由宿主处理）。
  • IO： FileReader/FileWriter 平台默认编码（注释写明与 UTF-8 命名差异，实际为平台 charset）。
  • 损坏文件： 扫描目录时解析失败的 .json 跳过，不中断列表。

响应与展示

业务逻辑
loadAllNotes()遍历 .json，解析、排序。
matchesKeyword(note, keywordLower)标题或正文包含（小写）；空关键词视为全匹配。
toSummary(JSONObject)生成列表行：id, title, createdAt, updatedAt, preview（无全文）。
createNote校验 title、写入新文件。
listNotes可选 keyword 过滤。
getNote必填 id，文件须存在。
updateNote必填 id，至少提供 title 或 content 键之一（可为空串）。
deleteNote删除文件，返回 deleted, id。
searchNotes必填非空 keyword。
exportNotepath 以 .txt 结尾则写该文件；否则视为目录，写入 {id}.txt；内容第一行标题，空行后正文。

User / LLM → ModuleRuntime.invokePlugin("notes", action, paramsJson) → NotesPlugin.invoke → 读写 /sdcard/PandaGenie/data/notes/*.json → ok(outputString, _displayText) → 前端展示列表/摘要或导出路径
