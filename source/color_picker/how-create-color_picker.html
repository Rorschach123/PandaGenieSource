模块概述

本模块为纯计算型插件：Context 不参与颜色运算。提供 HEX / RGB / HSL / CMYK 互转、CSS 命名色表解析、rgb()/hsl() 函数解析、和谐色板（complementary、analogous、triadic、split-complementary、tetradic/square）、SecureRandom 随机色，以及最近 CSS named color 匹配（RGB 欧氏距离）。

架构与数据流

NativeBridge.invoke(action, params) → ColorPickerPlugin.invoke() → handleConvertColor | handleGeneratePalette | handleRandomColor | handleGetColorInfo → JSONObject 业务结果 ← { success, output: <JSON string>, _displayText }

注意：ok(JSONObject output, String displayText) 将 output 以 字符串形式置于外层 output 字段（与部分插件一致）。

颜色数学（实现摘要）

  • RGB → HSL： 标准 sRGB 归一化后求 max/min，H∈[0,360)，S/L∈[0,100]。
  • HSL → RGB： 分段 hueToRgb 插值。
  • RGB → CMYK： K = 1 - max(R,G,B)，再求 C/M/Y 分母 (1-K)。全黑边界返回 CMYK = 0,0,0,100。
  • 最近命名色： findClosestNamed 遍历 NAMED_HEX，colorDistanceSq 为 RGB 差平方和，返回 sqrt 作为 distance。

API Actions — 输入 / 输出

convertColor
参数说明
input必填，颜色字符串
fromFormat默认 auto；可选 hex|rgb|hsl|cmyk
toFormat默认 all；或 hex|rgb|hsl|cmyk，此时额外字段 result 为单一格式字符串

auto 解析顺序： 灵活 HEX → 逗号 RGB/HSL/CMYK → rgb()/rgba() 前三个整数 → hsl()/hsla()（百分比/0–1 小数兼容）→ NAMED_HEX 表。

toFormat=all： buildColorJson 含 hex, rgb, rgbCss, hsl, hslCss, cmyk, 分量 r,g,b,h,s,l,c,m,y,k。

generatePalette

baseHex 经 normalizeHexParam；harmony 默认 complementary，下划线会替换为连字符。splitcomplementary 与 split-complementary 等价；square 与 tetradic 等价。

output： baseHex, harmony, colors（HEX 字符串数组，含 # 由 rgbToHex 拼接）。

randomColor

count：默认 1，clamp 1–20。hueMin/hueMax 默认 0/360，若颠倒则交换。饱和度约 55–95%，亮度约 45–70%。

output： colors, count。

getColorInfo

参数 hex 或备用键 input。output： hex, closestName, closestHex（带 #）, distance（四位小数）。
