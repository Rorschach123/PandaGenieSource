模块概述 Module overview

贪吃蛇（snake_game） 在插件进程内维护一局游戏的内存状态：蛇身坐标列表、食物坐标、当前方向、得分、gameOver、gameStarted、难度 difficulty（1–3）。 通过 ModulePlugin#invoke 暴露 4 个 action：startGame、move、getState、restart。 棋盘大小固定为 20 × 15 单元格（宽 × 高），与源码常量 BOARD_WIDTH、BOARD_HEIGHT 一致。

棋盘与难度 Board & difficulty

  • Board： BOARD_WIDTH = 20，BOARD_HEIGHT = 15。渲染字符：空地 ·，蛇头 ●，蛇身 ○，食物 ★，带边框 Unicode 线框。
  • 难度： startGame(difficulty) 将参数限制在 [1,3]。得分：score += 10 * difficulty（每吃一个食物）。
  • 难度名称（_displayText）： 简单 / 普通 / 困难 对应 1 / 2 / 3。

状态机与碰撞检测

游戏流程
  1. startGame → initGame()：蛇初始 3 节，居中偏左水平向右；spawnFood() 随机生成不与蛇重叠的食物。
  2. move(direction)：若未 startGame 则 error；若已 gameOver 则仍返回当前 state（success）。
  3. 方向：若传入空串则沿用当前方向；禁止 180° 掉头（与当前方向相反则忽略新方向）。非法方向字符串返回 error。
  4. 计算新头坐标 (newX,newY) 后：先判撞墙（越界）→ gameOver；再在插入新头之前将 (newX,newY) 与当前整条蛇各节比较 → 重合则 gameOver；随后 snake.add(0, {newX,newY})；若吃到食物则加分并 spawnFood 且不去尾，否则移除尾节。
  5. 吃到食物：得分 += 10 * difficulty，并重新生成食物。

四大 API Actions

action输入 params行为
startGamedifficulty 可选，默认 2新开局并返回 state + _displayText
movedirection：up/down/left/right 或空（保持方向）单步移动；撞墙/撞身则 gameOver
getState无返回当前 state；未开始则 error
restart无保持 difficulty，重新 initGame

方向键提示在 _displayText 中：up / down / left / right。

game state JSON（output 字段内嵌字符串）

成功时 output 为 JSONObject.toString()，主要字段：

{
  "gameStarted": true,
  "gameOver": false,
  "score": 0,
  "difficulty": 2,
  "direction": "right",
  "boardWidth": 20,
  "boardHeight": 15,
  "snakeLength": 3,
  "snake": [ {"x":0,"y":0}, ... ],
  "food": { "x": 0, "y": 0 },
  "board": "┌──…──┐\\n│…│\\n└──…──┘"
}

数据流 Data flow

Client 调用 invoke(action, paramsJson) → 修改 SnakePlugin 内存状态（单例插件生命周期内持久） → buildState() → output 字符串 → formatDisplay() → _displayText（ASCII/Unicode 棋盘 + 中文说明）

注意： 状态保存在插件实例字段；宿主若卸载/重载插件类则状态丢失。
