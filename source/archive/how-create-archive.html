架构总览 · Architecture Overview

本模块采用 四层架构：

  1. HTML UI（index.html + common.css）：用户界面，通过 window.NativeBridge 与宿主通信。
  2. Java Plugin（ArchivePlugin 实现 ModulePlugin）：解析 action 与 JSON 参数，路径规范化，调用 JNI 封装类。
  3. JNI Bridge（ArchiveLib.java / ArchiveLib.kt + archive_jni.cpp）：Java/Kotlin native 方法与 C++ 符号绑定。
  4. C++ Native（ArchiveModule + miniz + zlib）：实际压缩/解压；sandbox 在 native 层通过 SandboxGuard 校验读写路径。

通信路径：WebView 中 NativeBridge.invoke(action, JSON.stringify(params)) → 宿主将请求交给 ArchivePlugin.invoke() → ArchiveLib JNI → archive::ArchiveModule → 文件系统。

安全模型：模块声明 MANAGE_EXTERNAL_STORAGE；运行时还可由宿主配置 ArchiveLib.nativeConfigureSandbox(readPaths, writePaths)，在 native 层 sandbox_guard.cpp 与 ZIP/TAR/GZ 各入口的 checkRead/checkWrite 共同限制可访问路径。（具体由 PandaGenie ModuleRuntime 与插件加载流程配合，不在本文展开。）

1. ArchivePlugin.java · Java 插件层

包名：ai.rorsch.moduleplugins.archive

实现：ModulePlugin

职责：所有归档操作的入口；解析 paramsJson；将 /sdcard 路径解析为真实外部存储根路径（resolveStoragePath）；压缩前校验输入存在且可读、创建输出父目录；将 native 布尔结果或 listContents 的 JSON 字符串格式化为统一 JSON 响应；可选 _displayText 供 UI 展示。

invoke(Context, String action, String paramsJson)

主分发器。空 paramsJson 经 emptyJson() 视为 "{}"。

compressZip

参数（JSON）：

  • inputPaths 或 input：String，逗号分隔的多个绝对路径。
  • outputPath 或 output：输出 .zip 路径。
  • password：可选；非空时 native 在生成标准 ZIP 后对整文件做额外封装（见 C++ 层）。

行为：validateCompressArgs → ensureParentDir → lib.compressZip(inputs, output, pwd)。

返回：成功时 success: true，output: "true"，_displayText 含路径与估算大小；校验失败 success: false + error 英文说明；native 失败 error 含 “failed in native code (check logcat tag=ArchiveModule...)”。

decompressZip

参数：archivePath 或 path；outputDir；password（可选）。

行为：解压到目录；密码与 C++ 中 ENC 标记解密流程对应。

返回：布尔结果 JSON + 成功时解压统计展示文案。

compressTar / decompressTar

compress：inputPaths/input，outputPath/output。无密码。

decompress：archivePath/path，outputDir。

compressGz / decompressGz

compressGz：inputPath/input，outputPath/output；单独校验非空与源文件存在。

decompressGz：archivePath/path，outputPath（单文件输出）。

compressTarGz / decompressTarGz

参数形式分别同 TAR 压缩与 TAR 解压；底层为先 TAR 再 GZ 或先 GZ 再 TAR（见 native）。

listContents

参数：archivePath 或 path。

行为：直接返回 lib.listContents(path) 的字符串（native 生成的 JSON 数组）。

返回：ok(raw, displayText) — success: true，output 为条目 JSON 字符串，_displayText 为截断列表展示。

辅助方法（实际源码）

方法作用
emptyJson(String)空参数转为 "{}"，避免 JSONObject 解析异常。
ok(String) / ok(String, String)成功响应：success、output，可选 _displayText。
boolResult(boolean, String action, String display)压缩/解压布尔结果；失败时写入 error 提示查看 logcat ArchiveModule。
error(String)success: false + error 消息。
validateCompressArgs / splitPaths / ensureParentDir压缩参数校验与目录创建。
resolveStoragePath/sdcard 前缀映射到 Environment.getExternalStorageDirectory()。
formatDecompressDisplay / formatCompressDisplay / formatListContentsDisplay人类可读 _displayText。
说明：用户文档中若提到 configureSandbox / clearSandbox 在 Plugin 内 — 当前仓库中它们位于 ArchiveLib.java 的 static native 方法 nativeConfigureSandbox / nativeClearSandbox，由运行时/宿主在适当时机调用，而非 ArchivePlugin 实例方法。

3. ArchiveLib.kt · Kotlin JNI 桥接

包名同为 ai.rorsch.pandagenie.nativelib。external fun 与 Java 版实例方法一一对应（Array<String> 对应 String[]）。

不包含System.loadLibrary（由 Java ArchiveLib 静态加载保证 so 已载入）。当前片段中亦未声明 nativeConfigureSandbox / nativeClearSandbox；若 Kotlin 侧需直接调用，应在 ArchiveLib.kt 中补充 external 与 @JvmStatic 等 JNI 约定并与 archive_jni.cpp 符号一致。

4. C++ Native 层

4.1 archive_module.h

  • 命名空间：archive
  • struct ArchiveEntry：name, size, compressedSize, isDirectory
  • class ArchiveModule：静态方法 compressZip, decompressZip, compressTar, decompressTar, compressGz, decompressGz, compressTarGz, decompressTarGz, listContents
  • 私有辅助：encryptDecryptBuffer（XOR + 位置相关混淆，用于“密码”封装）, addFileToTar, getBaseName, mkdirRecursive

4.2 archive_module.cpp（约 484 行）

  • ZIP：使用 miniz（mz_zip_writer_* / mz_zip_reader_*）生成与读取标准 ZIP。密码：非空密码时先在明文 ZIP 上调用 encryptDecryptBuffer，再写入 4 字节魔数 'E','N','C',1 前缀；解压时若提供密码则去前缀、解密、写临时文件再交给 miniz。注意：此为自定义混淆而非标准 ZIP AES/PKWARE 加密，与常规 ZIP 工具不兼容。
  • TAR：自定义 512 字节 TarHeader（USTAR magic），octalToStr / octalFromStr / tarChecksum。
  • GZ：zlib 的 gzopen / gzread / gzwrite / gzclose。
  • TAR.GZ：compressTarGz：先 outputPath + ".tmp.tar" 写 TAR，再 compressGz 到目标并删除临时文件；decompressTarGz 先 GZ 到临时 TAR 再 TAR 解压。
  • listContents：按扩展名；.zip 走 central directory；.tar 顺序读 512 头（.tar.gz 扩展名判断为 gz 而非 tar，故 listContents 主要覆盖 zip 与 tar，与实现一致）。
  • 各操作前调用 sandbox::SandboxGuard::checkRead / checkWrite。

4.3 archive_jni.cpp（约 156 行）

宏 Java_ai_rorsch_pandagenie_nativelib_ArchiveLib_* 映射到 archive::ArchiveModule。listContents 将 vector<ArchiveEntry> 手写拼接为 JSON 数组字符串（含 escapeJson）。nativeConfigureSandbox / nativeClearSandbox 委托 sandbox_jni::configureSandbox / sandbox_jni::clearSandbox（头文件 sandbox_guard_jni.h）。

4.4 miniz.h / miniz.cpp

精简 ZIP 实现，基于 zlib deflate/inflate；结构体 mz_zip_file_stat、mz_zip_archive；Reader：init_file, get_num_files, file_stat, extract_to_file, end；Writer：init_file, add_file, finalize_archive, end。

4.5 sandbox_guard 集成

CMakeLists.txt 将 shared_native/sandbox_guard.cpp 链入；archive_module.cpp 包含 sandbox_guard.h，所有读写在打开文件前经 SandboxGuard 校验。

5. CMakeLists.txt

  • add_library(archive_module SHARED ...)：源文件 archive_module.cpp, archive_jni.cpp, miniz.cpp, ../../shared_native/sandbox_guard.cpp
  • target_include_directories：include、shared_native
  • target_link_libraries：log、z（zlib）

API 参考表 · API Reference

下列 9 个 action 与 manifest.json / UI 一致。返回格式以 ArchivePlugin 实际输出为准。

compressZip

项目内容
描述将多个文件压缩为 ZIP（Chinese：ZIP 压缩；English：ZIP compression）。可选密码会对整包做 native 自定义封装。
输入参数inputPaths 或 input（String，逗号分隔）；outputPath 或 output；password（可选 String）
返回 JSON成功：{"success":true,"output":"true","_displayText":"..."}；失败：success:false + error
示例
NativeBridge.invoke("compressZip", JSON.stringify({
  inputPaths: "/sdcard/a.txt,/sdcard/b.txt",
  outputPath: "/sdcard/out.zip",
  password: "secret"
}));

decompressZip

描述解压 ZIP 到目录（Extract ZIP archive）。
输入参数archivePath 或 path；outputDir；password（可选）
返回 JSON布尔结果形式 + _displayText（解压文件数统计）
示例
NativeBridge.invoke("decompressZip", JSON.stringify({
  archivePath: "/sdcard/x.zip",
  outputDir: "/sdcard/out",
  password: ""
}));

compressTar

描述打包为 TAR（POSIX USTAR 风格 512 字节块）。
输入参数inputPaths/input；outputPath/output
返回 JSON同 compressZip 布尔分支（无密码字段处理）
示例
NativeBridge.invoke("compressTar", JSON.stringify({
  inputPaths: "/sdcard/doc.txt",
  outputPath: "/sdcard/doc.tar"
}));

decompressTar

描述解压 TAR 到目录。
输入参数archivePath/path；outputDir
返回 JSON布尔结果 + 展示文案
示例
NativeBridge.invoke("decompressTar", JSON.stringify({
  archivePath: "/sdcard/a.tar",
  outputDir: "/sdcard/extract"
}));

compressGz

描述gzip 压缩单个文件。
输入参数inputPath/input；outputPath/output
返回 JSON布尔结果；空路径或文件不存在返回 error 英文消息
示例
NativeBridge.invoke("compressGz", JSON.stringify({
  inputPath: "/sdcard/big.bin",
  outputPath: "/sdcard/big.bin.gz"
}));

decompressGz

描述gzip 解压为单文件。
输入参数archivePath/path；outputPath
返回 JSON布尔结果；formatDecompressDisplay(..., true) 单文件模式
示例
NativeBridge.invoke("decompressGz", JSON.stringify({
  archivePath: "/sdcard/f.gz",
  outputPath: "/sdcard/f.out"
}));

compressTarGz

描述先 TAR 再 gzip（.tar.gz）。
输入参数inputPaths/input；outputPath/output
返回 JSON布尔结果 + 压缩展示
示例
NativeBridge.invoke("compressTarGz", JSON.stringify({
  inputPaths: "/sdcard/a.txt,/sdcard/b.txt",
  outputPath: "/sdcard/ab.tar.gz"
}));

decompressTarGz

描述gunzip 得临时 TAR 再解压到目录。
输入参数archivePath/path；outputDir
返回 JSON布尔结果 + 展示文案
示例
NativeBridge.invoke("decompressTarGz", JSON.stringify({
  archivePath: "/sdcard/pkg.tar.gz",
  outputDir: "/sdcard/pkg"
}));

listContents

描述列出归档内条目（实现侧对 .zip / .tar 扩展名分支；archive_jni 输出含 name/size/compressedSize/isDirectory）。
输入参数archivePath 或 path
返回 JSON{"success":true,"output":"[{\"name\":\"...\",\"size\":...,\"compressedSize\":...,\"isDirectory\":false}, ...]","_displayText":"..."}
示例
NativeBridge.invoke("listContents", JSON.stringify({
  archivePath: "/sdcard/sample.zip"
}));
