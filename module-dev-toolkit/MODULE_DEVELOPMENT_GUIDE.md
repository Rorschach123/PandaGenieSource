## .mod 文件结构 / .mod File Structure

`.mod` 本质上是一个 zip/jar 兼容包，打包完成后的典型结构：

A `.mod` file is a zip/jar-compatible package. Typical structure after packaging:

```text
example-1.0.mod
├─ manifest.json          ← 含 devCertFingerprint（自动注入）/ with devCertFingerprint (auto-injected)
├─ index.html             ← H5 页面入口 / H5 page entry
├─ common.css             ← 公共样式 / Common styles
├─ plugin.jar             ← 编译后的插件 / Compiled plugin (DEX format)
├─ libs/
│  ├─ arm64-v8a/
│  │  └─ libexample.so
│  └─ armeabi-v7a/
│     └─ libexample.so
└─ META-INF/
   ├─ DEV.SF / DEV.RSA           ← 开发者签名 / Developer signature
   ├─ OFFICIAL.SF / OFFICIAL.RSA ← 官方签名 / Official signature
   └─ MANIFEST.MF
```

| 文件 / File | 说明 / Description |
|-------------|-------------------|
| `manifest.json` | 模块元数据。打包时自动注入 `devCertFingerprint` 字段 / Module metadata. `devCertFingerprint` is auto-injected during packaging |
| `index.html` | 模块 H5 页面入口 / Module H5 page entry |
| `common.css` | 公共样式文件（可选）/ Common stylesheet (optional) |
| `plugin.jar` | 由 Java 源码编译并转 DEX 后生成 / Compiled from Java source, converted to DEX |
| `libs/<abi>/*.so` | 原生库，按 ABI 分目录。无原生库的模块可省略 / Native libs by ABI. Omit if none |
| `META-INF/*` | jarsigner 生成的双重签名 / Dual signatures generated by jarsigner |

### capabilities 沙箱能力声明 / Sandbox Capabilities

`capabilities` 声明模块需要的系统能力。APP 在执行模块任务前会根据此声明进行权限检查，用户可在「设置 → 权限管控」中配置每个模块的权限。

`capabilities` declares what system capabilities the module needs. The APP checks these before executing any module task. Users can configure permissions per module in "Settings → Permissions".

| 值 / Value | 说明 / Description |
|------------|-------------------|
| `file_read` | 读取文件 / Read files |
| `file_write` | 写入文件 / Write files |
| `network` | 网络访问 / Network access |
| `camera` | 使用相机 / Camera access |
| `location` | 获取位置 / Location access |
| `microphone` | 使用麦克风 / Microphone access |
| `clipboard` | 读写剪贴板 / Clipboard access |

**重要 / Important**:
- 未声明 `capabilities` 的模块不受沙箱限制 / Modules without `capabilities` are not sandboxed
- 声明了 `file_read` 或 `file_write` 的模块，用户必须在权限管控中配置允许的路径才能执行 / Modules declaring `file_read` or `file_write` require the user to configure allowed paths in Permissions
- 权限检查在插件代码执行**之前**由主体 APP 完成，模块代码无法绕过 / Permission checks happen **before** plugin code executes; modules cannot bypass them

**示例 / Example**:

```json
"capabilities": ["file_read", "file_write"]
```

无特殊能力需求的模块（如纯计算）：

Modules with no special capability needs (e.g. pure computation):

```json
"capabilities": []
```

## H5 页面多语言 / H5 Page i18n

模块的 `index.html` 也应支持中英文。APP 会在 WebView 加载完成后注入全局变量 `_lang`（值为 `"zh"` 或 `"en"`），并调用 `applyI18n()` 函数。

Module `index.html` should also support Chinese and English. The APP injects a global `_lang` variable (`"zh"` or `"en"`) into the WebView after page load, and calls `applyI18n()`.

### 推荐实现方式 / Recommended Implementation

```html
<script>
// 语言检测：优先使用 APP 注入的 _lang，否则根据浏览器语言判断
// Language detection: prefer APP-injected _lang, else detect from browser
var _lang = (typeof _lang !== 'undefined') ? _lang :
    (navigator.language && navigator.language.startsWith('zh') ? 'zh' : 'en');

function T(zh, en) { return _lang === 'zh' ? zh : en; }

function applyI18n() {
    document.querySelectorAll('[data-zh]').forEach(function(el) {
        el.textContent = _lang === 'zh' ? el.getAttribute('data-zh') : el.getAttribute('data-en');
    });
}
</script>

<!-- 静态文本使用 data-zh / data-en 属性 -->
<!-- Static text uses data-zh / data-en attributes -->
<h1 data-zh="计算器" data-en="Calculator">计算器</h1>
<button data-zh="计算" data-en="Calculate">计算</button>

<!-- 动态文本使用 T() 函数 -->
<!-- Dynamic text uses T() function -->
<script>
document.getElementById('result').textContent = T('计算结果', 'Result');
</script>
```

## 插件 Java 代码 / Plugin Java Code

每个模块需要一个实现 `ModulePlugin` 接口的 Java 类。

Each module needs a Java class implementing the `ModulePlugin` interface.

```java
package ai.rorsch.moduleplugins.example;

import android.content.Context;
import ai.rorsch.pandagenie.module.runtime.ModulePlugin;
import org.json.JSONObject;

public class ExamplePlugin implements ModulePlugin {
    @Override
    public String invoke(Context context, String action, String paramsJson) throws Exception {
        JSONObject params = new JSONObject(paramsJson != null ? paramsJson : "{}");
        switch (action) {
            case "hello":
                String name = params.optString("name", "World");
                return new JSONObject()
                    .put("success", true)
                    .put("output", "Hello, " + name + "!")
                    .toString();
            default:
                return new JSONObject()
                    .put("success", false)
                    .put("error", "Unsupported action: " + action)
                    .toString();
        }
    }
}
```

**要求 / Requirements**:
- 返回值必须是 JSON 字符串，包含 `success` 和 `output`（成功）或 `error`（失败）/ Return JSON string with `success` and `output` (success) or `error` (failure)
- 插件源码放在 `PandaGenieSource/source/<moduleId>/plugin_src/` 下 / Plugin source goes in `PandaGenieSource/source/<moduleId>/plugin_src/`
- 包名推荐 `ai.rorsch.moduleplugins.<moduleId>` / Recommended package: `ai.rorsch.moduleplugins.<moduleId>`

## 双重签名机制 / Dual-Signature System

每个 `.mod` 文件携带两层独立的 jarsigner 签名：

Each `.mod` file carries two independent jarsigner signatures:

```text
┌──────────────────────────────────────────────┐
│  .mod 文件 / .mod file                        │
│                                              │
│  manifest.json                               │
│    └─ devCertFingerprint: "DF27..."          │  ← 锁定开发者证书 / Locks dev cert
│                                              │
│  META-INF/                                   │
│    ├─ DEV.SF / DEV.RSA     (第一层 / Layer 1) │  ← 开发者身份 / Developer identity
│    └─ OFFICIAL.SF / OFFICIAL.RSA (第二层 / L2) │  ← 官方认证 / Official certification
└──────────────────────────────────────────────┘
```

| 签名层 / Layer | sigfile | 用途 / Purpose | 必需 / Required |
|---------------|---------|---------------|----------------|
| 开发者签名 / Developer | `DEV` | 标识发布者身份，用于同 ID 冲突检测 / Identifies publisher, used for ID conflict detection | **是 / Yes (always)** |
| 官方签名 / Official | `OFFICIAL` | 证明经过官方认证 / Proves official certification | 非开发者模式下必需 / Required outside dev mode |

### 绑定关系 / Binding

```text
官方签名 → 保护 manifest.json → manifest 声明 devCertFingerprint → 必须匹配 DEV 证书
Official sig → protects manifest.json → manifest declares devCertFingerprint → must match DEV cert
```

### 密钥存储 / Key Storage

| 密钥 / Key | 路径 / Path | 初始化 / Init |
|-----------|------------|--------------|
| 开发者 / Developer | `Keystore/dev_signing/private/dev-keystore.p12` | `mk_module.ps1 -Action init-dev-signing` |
| 官方 / Official | `Keystore/module_signing/private/official-keystore.p12` | `mk_module.ps1 -Action init-signing` |

密码使用 Windows DPAPI 加密存储 / Passwords are encrypted with Windows DPAPI.

## 生成流程 / Build Process

`.mod` 的生成分为 9 步 / The `.mod` build process has 9 steps:

1. **编译 Android App** — 确保主工程类、JNI 输出和插件编译依赖已生成 / Build Android App to generate classes, JNI output, and plugin dependencies
2. **读取 manifest.json** — 获取模块元数据和版本号 / Read module metadata and version
3. **复制前端资源** — `manifest.json`、`index.html`、`common.css` / Copy frontend resources
4. **注入 devCertFingerprint** — 从开发者签名元数据读取 SHA-256 指纹，写入 `manifest.json` / Inject dev cert SHA-256 fingerprint into `manifest.json`
5. **编译插件** — `javac` → `jar` → `d8` (DEX) → `plugin.jar` / Compile plugin: `javac` → `jar` → `d8` (DEX) → `plugin.jar`
6. **收集原生库** — 按 ABI 拷贝 `.so` 到 `libs/` / Collect native libs by ABI into `libs/`
7. **压缩** — 将临时目录压缩为未签名包 / Compress temp directory into unsigned package
8. **开发者签名** — `jarsigner -sigfile DEV` / Developer signing
9. **官方签名** — `jarsigner -sigfile OFFICIAL`，输出 `modId-version.mod` / Official signing, output `modId-version.mod`

## 使用命令 / Commands

### 1. 初始化签名材料（首次）/ Initialize Signing (First Time)

## 动态加载机制 / Dynamic Loading

APP 不在 APK 中硬编码任何模块信息。所有模块能力完全由 `.mod` 包内的 `manifest.json` 声明。

The APP does not hardcode any module info in the APK. All module capabilities are declared in the `.mod` package's `manifest.json`.

1. APP 启动时扫描 `/sdcard/PandaGenie/modules/*.mod` / On startup, scans `/sdcard/PandaGenie/modules/*.mod`
2. 签名校验通过后解压并读取 `manifest.json` / After signature verification, extracts and reads `manifest.json`
3. 从 `apis` 动态构建 AI system prompt / Dynamically builds AI system prompt from `apis`
4. AI 根据 prompt 中的模块列表进行任务规划 / AI plans tasks based on the module list in the prompt
5. 用户说"打开xxx"时，AI 返回 `openModule`，APP 跳转到模块 H5 页面 / When user says "open xxx", AI returns `openModule`, APP navigates to module H5 page

## 开发者模式 / Developer Mode

设置中有"开发者模式"开关 / There is a "Developer Mode" toggle in Settings.

- 默认关闭 / Off by default
- 可临时开启 10分钟、1小时、1天 / Can be temporarily enabled for 10 min, 1 hour, 1 day
- 开启后允许加载非官方签名的第三方模块 / Allows loading third-party modules without official signature
- 超时后自动关闭 / Auto-disables on timeout
- **开发者模式只豁免官方签名检查**，以下检查始终执行 / **Only exempts official signature check**; these always apply:
  - 开发者签名必须存在 / Developer signature must exist
  - `devCertFingerprint` 绑定校验 / `devCertFingerprint` binding verification
  - 同 ID 模块的开发者签名冲突检测 / Same-ID module developer signature conflict detection
- 使用 `-SkipSigning` 生成的完全未签名模块，开发者模式下也**无法加载** / Fully unsigned modules (via `-SkipSigning`) **cannot be loaded** even in dev mode

## 安全保障 / Security

| 攻击场景 / Attack Scenario | 防御机制 / Defense |
|---------------------------|-------------------|
| 替换 DEV 签名 / Replace DEV signature | `devCertFingerprint` 不匹配 → 拒绝 / Fingerprint mismatch → rejected |
| 修改 manifest 中的指纹 / Modify fingerprint in manifest | 官方签名被破坏 → 拒绝 / Official signature broken → rejected |
| 删除 DEV 签名 / Remove DEV signature | 检测到缺少开发者签名 → 拒绝 / Missing dev signature detected → rejected |
| 完全无签名 / No signature at all | 签名校验失败 → 拒绝 / Signature verification fails → rejected |
| 不同开发者冒用同一 ID / Different dev using same ID | ID + 开发者指纹冲突 → 拒绝 / ID + fingerprint conflict → rejected |

### 模块加载规则 / Module Loading Rules

| 场景 / Scenario | 行为 / Behavior |
|----------------|----------------|
| 缺少开发者签名 / Missing dev signature | **拒绝加载 / Rejected** (always) |
| `devCertFingerprint` 不匹配 / Fingerprint mismatch | **拒绝加载 / Rejected** |
| 双签名完整 / Both signatures valid | 正常加载 / Normal load |
| 仅开发者签名 / Dev signature only | 需开发者模式 / Requires dev mode |
| 同 ID 同开发者指纹 / Same ID, same dev fingerprint | 覆盖安装 / Overwrite install |
| 同 ID 不同开发者指纹 / Same ID, different dev fingerprint | **拒绝加载 / Rejected** |
